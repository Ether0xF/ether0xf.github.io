貓的體重~SP~

发表于 2013-05-24   |   分类于 Notes   |  

前幾日那個『貓的體重』的題目，幸得Ellena的評論提醒發現了一處錯誤。
果然是腦子鈍了。當時雖然想到了用係數(k1+k2)<1來判斷無窮生長，但是沒有反應過來這其實是個無窮遞縮數列。
因此公比(k1+k2)<1應該為|k1+k2|<1。
加上貓的體重允許負數，因此判斷無窮生長的條件更加複雜一些。將判斷方法略做修改。
不得不說，體重允許負數這個條件真是比想象中噁心。

代碼新改動了判斷貓的體重增長的方法weightGrow()，根據以下公比的不同情況進行判斷。
然後為了代碼的美觀，將成長天數遞增的循環單獨寫了一個方法 int dayPlus(int dayin)。新的weightGrow()各種條件語句的嵌套和層層縮進已經快要觸發我的強迫症因此我決定稍微減少一下嵌套和縮進。
最後為了方便測試，改善輸出樣式以求美觀可讀。
順便說句iTerm的背景圖片果然還是放刺客信條帶感。

關於等比數列公比的內容看這裏：Geometirc Series#Common Ratio

要使貓的體重無窮增長，且因為有體重限制(upper bound)，則要使等比數列totalWeight(i + 1) = (k1+k2) * totalWeight(i)無窮遞縮。由於有體重限制和負體重的情況，這個等比數列和普通無窮等比數列的不同就是，不討論收斂(converge)和發散(diverge)以及極限(limit)的情況。
換言之，逐日的體重能無窮遞縮就行。
由於貓的體重可以為負數，在第一天的初始體重weight(1) ≤ k的大前提下（否則第一天就不會繼續生長，立刻停止）要分為三種情況來討論公比的各種情況。（等我再去扒拉一張草稿紙來……

第一種情況：k >0 這個屬於一般情況，無窮遞縮數列種所有項ai∈R且所有ai的和不大於體重限制k。（按照這隻貓的尿性體重為無理數應該也是妥妥的沒問題）
因此公比r = (k1 + k2)分以下情況：

  • |r| < 1 i.e. -1 < r < 1 數列各項無限向0收斂。inf。
  • r > 1 當weight(1) > 0時，體重最終超過限制k，停止生長。當weight(1) < 0時，數值向負無窮發散。inf。
  • r < -1 體重在正負間振盪並最終超過限制k。停止生長。
  • r = 1 奇葩的情況其一，體重會無限地保持初始值。由於有weight(1)≤k的大前提，這個該算是已經停止生長了呢，還是算是無限呢……這裏視作inf。

  • r = -1 奇葩的情況其二。體重會在±firstDayWeight之間振盪（oscillation，想了很久不知道這個詞在中文裏怎麼說，感謝123君指點，最終敲定了振盪這個詞），但是由於有大前提。inf。
    第二種情況：k ≤ 0 and weight(1) < 0 很容易引起問題的一個情況。特別特別討厭……需要一些仔細的考慮。
    因此公比r = (k1 + k2)分以下情況：

  • -1 < r < 0 雖然會向0收斂但是因為負負得正，所以會超過限制k。停止生長。

  • 0 < r < 1 陷阱。雖然數列無限向0收斂，但是要視k的情況而定。
    當k < 0時，停止生長。
    當k = 0時，inf。
  • r > 1向負無窮發散。inf。
  • r < -1 體重在正負間振盪並最終超過限制k。停止生長。
  • r = 1 奇葩的情況其一，體重會無限地保持初始值。由於有weight(1)≤k的大前提，這個該算是已經停止生長了呢，還是算是無限呢……這裏視作inf
  • r = -1 奇葩的情況其二。體重會在±firstDayWeight之間振盪，但是由於有大前提。inf。
    第三種情況：**weight = 0 和 r = 0
    這個情況是奇葩中的霸王花。由於初始體重為0，因此不管係數r = (k1+k2)為何，體重都不會有一絲一毫的變化，一直為0。且由於大前提weight(1)≤k，所以勢必有k > 0 。於是這個情況就像r = 1**一樣，視作inf。相同，由於r = 0，不管初始體重為何都會變為0。不過，這個只在k ≥ 0時為inf。當k < 0時，停止生長。

代碼在第二頁，測試和輸出在第三頁。

[java]
//new on 5-24
public String weightGrow(){
int day = 1;
float r = k1+k2;

    if(weight(1) &lt;= k){

        if (weight(1) == 0 || Math.abs(r) == 1.0)
            return &quot;inf&quot;;
        //------------if k &gt; 0-----------------
        else if (k &gt; 0){
            if (Math. abs(r) &lt; 1.0 || r == 0)
                return &quot;inf&quot;;
            else if (r &gt; 1.0){
                if (weight(1) &gt; 0){
                    day = dayPlus(day);
                }
                else if (weight(1) &lt; 0)
                    return &quot;inf&quot;;
            }
            else if (r &lt; -1.0){
                day = dayPlus(day);
            }
        }
        //-------------if k &lt; 0-----------------
        else if (k &lt;= 0){
            if (r == 0){
                if (k == 0)
                    return &quot;inf&quot;;
                else if (k &lt; 0)
                    day = dayPlus(day);
            }
            if (r &gt; -1.0 &amp;&amp; r &lt; 0)
                day = dayPlus(day);
            else if (r &gt; 0 &amp;&amp; r &lt; 1.0){
                if (k == 0.0)
                    return &quot;inf&quot;;
                else if (k &lt; 0)
                    day = dayPlus(day);
            }
            else if (r &gt; 1.0)
                return &quot;inf&quot;;
            else if (r &lt; -1.0)
                day = dayPlus(day);
        }
    }

    if (day == 1)
        return Integer.valueOf(day).toString();
    else
        return Integer.valueOf(day - 1).toString();

}
