Type of R Objects

vector
: 最基本的type。所有R objects都是vector。
有五种六种基本类型(atomic vector types): character, numberic, integer, complex, logical, raw
是的，课程和书上告诉你有五种，但是cran的文档告诉你，有六种。
vector中所有元素都是同类型。
必须是，不是的情况下会转换类型。

list
: 泛型vector，所有元素不必是同一类型。

array
: 带属性dim（维度）的vector。
matrix就是一个二维array。

data.frame
: 基本且常用的，同时具有matrix和list的特性的数据结构。
每一列都相当于一个变量列表（每一行代表一个变量）。column-wised且所有列具有相同的长度。
行的名是唯一的，没有名字时会自动生成行号代替名字。
列的名字可以重复但是不能为空。

在抽取子集（尤其是从data.frame中抽取子集）之前，一定要先弄清楚这些type的区别和特点，记住什么样的操作符返回的类型到底是什么。往往是第一节课觉得类型这个很简单，结果实际操作的时候碰到类型Error会非常困惑。
尤其使用is.vector()，is.list()之类的函数时会发现subsetting的一些tricks。虽然vector是R Object中最基本的类型，但是并不是所有object的is.vector()都会返回TRUE。

Subsetting

Subsetting最重要的是掌握indexing的操作符的区别。
有三种：[,[[,$
[能用于一次性选择多个元素，而[[和$只能选择一个元素；
[[和$返回的都是atomic vector类型；
[[会去掉元素的names属性，而[可以返回recursive结构且保留names属性。
$的特殊之处在于只能使用字符串索引，只能作用于recursive object，如果元素的names属性为空，则不能用$操作符来获取此元素。
[和[[的索引值可以是字符，也可以是数字和逻辑值。

而data.frame可以同时像list和matrix一样indexing，但[和[[操作符返回的结果往往会背叛你想当然的结果。因为[，[[和$都是泛型，允许添加class method。data.frame在调用这些操作符的时候，实际上调用的是class method，而class method的行为并不一定要和basic method完全一致。

总结

上面就三种操作符在不同类型中的使用区别讲了很多细节，但是并不打算这样来总结。对于三种操作符的返回类型，实际上只要记住两个概念就好：atomic or recursive。

是atomic的R objects:

  • 上面提到的六种基本类型(atomic vector types)
  • NULL

除了names，S4类型的objects，外部指针，还有上面两种atomic objects以外，大多R objects都是recursive。

vector是atomic，array是有dim属性的vector所以也是atomic。
list是recursive。

recursive object调用[返回的值也是recursive，$只能被recursive object调用，和调用[[一样返回的是atomic objects.

atomic object不管调用[还是[[差别不大，只有array会调用特别的[方法来决定是否去掉dim属性。

atomic和recursive，只有牢记这两个概念。
虽然R语言强调大多数objects都是vector，但是因为is.vector()和names属性的存在，平添了一个例外，让定义变得很容易混乱。就如同上面matrix中提到的例子一样，an object could be atomic but not a vector.
即使尝试对这部分进行整理，整理完后发现例外依然遍地存在，并不存在偷懒的分类记忆法，最好的方法还是熟练多记（反正不难，而且是基础中的基础）。
所以不要想太多。
