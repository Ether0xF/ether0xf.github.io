苦夏

发表于 2013-08-17   |   分类于 The Journey   |  

我又很久沒更新了。
所以只是來稍微除個草。

其實最近做了不少事情。
TOEFL複習得心理壓力有點大。每天是上課，做作業，做題，抽空給『青龍淚』更兩句。本來打算勤快地在10月前把第一部分寫完的。其實我覺得這個潛力還是有的。時間軸已經來到『升龍傳』前了。

期間通關了DragonAge II，然後今晚做完作業後把DLC「遺產」通關了。總的來說，過度作，不管劇情還是系統各方面都不如DAO。法師被削得慘不忍睹（擦淚）。結局簡直坑爹。可預見的一定會發生的聖殿騎士和法師的衝突，本來Champion是努力平衡雙方可以將衝突的傷害儘量降低的。然後白癡安德斯對著教會booom一發把所有人所有事情包括Champion在內都推下了懸崖。霍克是那麼相信同伴的，沒想到安德斯利用霍克干了一件這樣的事情。感覺最後霍克真是暴躁到想一腳踹飛安德斯的。
最後開戰前選擇是殺死安德斯，還是一起作戰，還是讓他離開。
想來想去還是，Just go.
深深感覺到霍克的悲哀。
最後Champion離開了柯克沃。是我也想走的。

看完了『姑獲鳥之夏』，『狂骨之夢』還沒開始看。只能說，真是好頂贊。
京極夏彥的白描手法實在太棒了。看完一本書感覺吸收了很多很多文力。
但我感覺到10月前看完書架下面所有的系列是不可能了。

七夕和小梅子去做了任務放了真橙。
七夕任務真意絕對是死情緣。跳崖任務略略難。其精髓乃是you jump, I just have a look.
因為做任務的人很多，凡是碰到搶電梯搶採集的情況都一直狂按H鍵才能搶到。
還有最後洛道的任務憑著多年茶館除奸懲惡任務搶怪練出來的功夫，一個罡風鏢法妥妥地搞定。
還有在持國的佛手手指上放真橙結果真橙掉了一半看不到了什麼的……so sad

最近睡得不好，經常性睡眠不足或者低血壓，導致工作效率和食欲都持續低下。但是並沒有見到我的脂肪被消耗掉。
於是為了恢復體力和減肥……開始鍛煉了。
苦夏。

幻夜

发表于 2013-07-19   |   分类于 藏書箱   |  


原名:幻夜
作者: [日]東野圭吾
译者: 李炜
出版社: 南海出版公司
出版年: 2009年9月
页数: 424
ISBN: 978-7-5442-4527-2
內容簡介：
　　我们只能走在幻夜的路上，即使四周明亮有如白昼，那也仅是假象。
就算与你共度的每个夜晚都是幻夜，我也愿为你化身为影，至死不渝！

地震之后，宛如人间炼狱的断壁残垣中，水原雅也借机杀了舅舅，却被一神秘女子当场目击，她答应为水原终生保守秘密。
他们相偕前往东京，然而等待他们的，却是从此再无一丝太阳的无边幻夜：凡是接近过她的人，都遭逢厄运；凡是触碰过她过去的人，都不知所踪……
她，到底是谁？

『幻夜』分了好幾天來看完。說實話看的過程中並沒有產生多少想一口氣看下去的衝動，反而只是想快點看完這本書而已。尤其看最後一部分的晚上頭痛得很，一邊強迫症一樣看完最後幾十頁，結果並說不上多爽心。
只有一點有趣的是，『幻夜』的宣傳語也說是東野圭吾的巔峰之作，也一樣和『白夜行』比較（雖然說這次真的是姐妹篇了），但編輯你們真的沒把『惡意』和『幻夜』的宣傳語寫串嗎。

總的來說，並不多有趣，也沒有感覺到介紹上所說不折不扣的絕望和血流成河的人性之惡。十分中心思想的那段「我们只能走在幻夜的路上，即使四周明亮有如白昼，那也仅是假象。」也出現得非常非常早，而且並沒有多貫穿全書，讓人連一點點「點中中心戳中痛處」的感覺都沒有。再者，「幻夜」怎麼也是水原雅也的幻夜，和新海美冬其實沒有太大的關係。新海美冬再怎麼有模有樣地說出來都讓我不信。而且水原雅也明明是有白晝的只是他不信。更別說什麼的「我也愿为你化身为影，至死不渝！」，明明是沒有的事！我讀書少不要誑我！
看完後匆匆的在豆瓣上點了個五星然後趕緊蒙頭休息。第二天起來順手就改成了四星，然後在心裏默默加上一句，愚蠢的男人。
如果複習過『白夜行』，也許順手就改成三星了也說不定。

以上是三個月前的……日誌草稿。（何……
那天我一邊折騰著驅動把系統從WIN8裝成了WIN7，一邊寫了一半這個。
然後現在我又把WIN7折騰回WIN8了（……）等待消食的時候（……）挖出來繼續寫掉（……）。
況且當時看完總覺得整個人的狀態怪怪，現在來回想一下這本書，細節可能有些淡忘了，不過感覺上反而更易表述了。

作為和『白夜行』有千絲萬縷的關係的一作，實在難以擺脫被拿來和『白夜行』比較的命運。然而比較的話，在觀感和餘味上，『幻夜』始終還是遜了『白夜行』一些。也許是『幻夜』太膩，不像『白夜行』那樣苦澀清淡但餘味良久。
這很大原因也出在兩作的女主角的對比上。雖然美冬是否就是雪穗本人，作者沒有明示。不過網上對於兩人的身份的考證仍然數不勝數。其實不管是否將她們視作同一人——是將『幻夜』視作『白夜行』的續作，還是不一樣的兩作——都十分可玩味。

雪穗像是覆雪荒原的凍土上挺立的白花；有莖在凍土地下同根雙生，那大概是亮司。兩人都生存在冰冷的白夜中。很難說荒原和這兩人，誰更嚮往陽光，誰從誰這那裡汲取白夜的養料。不管是荒原還是白花和根莖，都讓這個故事像冰塊一樣透著冷意。越是打磨，冰塊越是透明，真相越是明朗，但冰塊的邊緣也越是鋒利見血。

相比下美冬更像是黑泥沼裏盛開的奇葩。光鮮亮麗，散發著誘惑獵物的迷魂香，然後將獵物拖入沒有根底的黑泥沼中。越是掙扎，陷得越深。雅也之於美冬是獵物，絕不是任何共生互生的關係。小說裏數次強調著美冬那種吸引人的危險莫名的魔性，都擺明這是捕獵者和獵物的關係。可惜的是，不管多少次明文強調，反而讓人對此的感覺更淡薄。遠不像『白夜行』，融化在故事裏的冷意貫穿始終。
這可能是『幻夜』相比『白夜行』的敗筆。甚至已經有了探索真相的偵探角色雅也，還要在將近尾幕的時候插入刑警角色加藤，讓場面略為混亂。尤其餘味不佳的便是雅也和刑警加藤最後的對決。原本打算殺死美冬的雅也，反而對捕獵者衍生出了獨佔欲，開始了和獵物同士之間的自相殘殺。無疑最後的受益者是美冬。空氣中甚至聞不到雅也和加藤爭鬥的硝煙和血腥味，而是……什麼味道都沒有。以美冬站在遊船甲板上為終景，故事用一種很彆扭的方式結局了。
沒有留下任何東西，沒有『白夜行』最終那種冷冷的苦澀味。至少雪穗和亮司懷抱著某種黑暗的感情和秘密，因此無法回到陽光底下。而美冬是全然沒有根底的，那種罪惡連根源都尋不到，徒有的光鮮的表像，因此才談不上餘味。

如果將『幻夜』視作『白夜行』的某種後續的話，失去亮司這一抹陽光，失去了寄託、依靠和顧忌、底線，雪穗變成了美冬，「白夜」變成「幻夜」，又讓人覺得唏噓和可怕。也許美冬將雅也當成亮司；也許不斷變得年輕的美冬，像回到雪穗的年齡，將自己的時間定格在有亮司的過去中。這反而品嘗出了些許光鮮表像下的空虛。而雅也無疑是可悲的犧牲品，加藤和諸多故事的過路人也是無辜的被捲入者。魔性不再是空談，反而成為了美冬對失去「白夜」的陽光、變成「幻夜」的世界的報復，是一場不可抵擋的復仇的大浪，將周圍的一切不顧後果地全數吞沒。這也許是才是雪穗冷面下的思念，美冬笑容下的空虛，看似完美的漁翁之利，實際上收穫的是無盡的不自知的悲哀。

然而東野圭吾卻說過，「不想讓『幻夜』成為『白夜行』的續集」。兩部作品都讀完的讀者確實是「開心地徜徉於各種各樣的想像」，只不過這樣的想像反而是基於「『幻夜』是『白夜行』的續集」的基礎上的。
這該說是作者的精心策劃呢，還是實際上的不如意的敗筆呢。
不可知。

總之，『幻夜』不會勾起我任何再讀第二遍的興趣，畢竟讀起來十分不舒服。
反而讀完『幻夜』後，對『白夜行』有種懷念感。這似乎從側面，越發捧高了『白夜行』的地位。

玩了一晚上编辑器

发表于 2013-07-06   |   分类于 Notes   |  

其實現在的內容就是在vim裏用vimrepress插件寫的。

雖然說只是想順便地熟悉vim。
因為手感實在太好了。

不过为了成功设置xmlrpc的error还是稍微折腾了我一下。

其實一開始打算學習Emacs。因為Lisp的大名如雷貫耳。加上Emacs又人稱偽裝成操作系統的編輯器（還有經典的煮咖啡功能，当然并不是用超文本咖啡壶控制协议<(=￣ˇ￣=)>），聽起來實在是狂霸酷拽屌炸天。

但是實際用起來以後，覺得自己像在玩avg一樣……整個晚上讀tutorial的同時就在不停按ctrl和Meta鍵。雖然說Emacs的快捷鍵和UNIX終端下的快捷鍵很多是一樣的，但是一直這樣用真的非常反健康。
畢竟……我上三個月的時候就已經確實變成一個腕管綜合徵患者了——簡單來說，得了鍵盤鼠標手——而且還是雙手。
全都多虧一直以来握壓感筆太緊張和這幾個月玩劍三的成果.因為我的劍三的鍵位不是數字鍵，全都是用字母的，其中小指用得最多，因為那個鍵位是半自動宏和主要攻擊技能的鍵位。

長年累月造成了現在悲劇……

當然這並不是說Emacs不好。說出這種話題就真怕會引發編輯器的口水戰。因為連Emacs的門都沒完全走進去，實在不敢評論Emacs如何如何。但是又不想像別的Emacs用戶一樣將ctrl的快捷鍵全都映射到Caps Lock上。

所以大概在Tutorial讀到64%的時候在喵喵姐的建議下安裝了MacVim.

hjkl移動光標的方式真是很爽快。非常簡便動感，像玩ACT遊戲一樣。
玩劍三的時候也有朋友說過我這個鍵位如果是別人上號的話根本不會用……但是真的非常靈活好用。有人看到後誤以為我是PVPer，其實我是純得不能再純的PVEer。

沒錯，就是這樣。Vim用起來就像在玩ACT遊戲一樣富有節奏感。看起来十分复杂，实际上完全是用手指肌肉去记忆的动作，一下子就能上手了。唯一的困難就是快捷鍵组合實在太多了，多到已經達到能搓招的程度了——那已經是FTG遊戲，不是我的守備範圍了。
不過作為默認的光標移動鍵位之一（雖然方向鍵也可以，但是和help.txt建議的一樣，方向鍵會讓手離開打字位置，變得不便），在insert mode和normal mode之間切換的時候實際上手指還是需要移動一個鍵位。
今晚不知道有多少次忘記移動就直接按下去了，導致h鍵在我的腦海裏一直沒有什麼存在感。

如果能把光標鍵改成jikl，insert mode toggle改成h多好。又直觀又不需要挪動鍵位（玩遊戲改鍵位的經驗）。

不過還是等之后再来研究key mapping和插件等諸多問題。

順便一說編輯器之戰實在是太有趣了。從旁觀者和outsider的角度來看，就像人世百態縮影的一種一樣。

Codecademy的Ruby Tutorial告一段落後，稍微有點迷茫。我對編程的學習總是很容易止步於不知道該做一個什麼小project來練手。目前大概只能慢慢補足算法的學習。

問題的描述很簡單。

  1. 一共有N個人圍成一圈。
  2. 從第一個人開始，數到K的時候將K位置上的人出列。
  3. 從K+1開始重新數。
  4. 重複2和3步驟，直到剩下一個人。
    求：最後一個人在原來隊列中位置，以及出列的順序。

分別用循環和遞歸兩種方法來做。語言是Ruby。

用循環的方法來模擬這個過程非常簡單。
思路大概是這樣的。

新建一個列表a，長度為N。

計數器+1

如果 計數器 等於 K

++將K位置移出隊列。

++計數器=0

++出列計數器＋1

++輸出被移除的位置

指針移到K+1位置

如果 指針 指向了列表的末端

循環直到隊列為空。輸出的最後一順位即為最後留下的人的原位置。

def josephusI(players, count_off)
  # set each element to be 1
  player_array = Array.new(players) { |element| element = 1 }
  count_out = 0
  counts = 0
  arr_index = 0

  until count_out > players - 1
    counts += player_array[arr_index] if player_array[arr_index] == 1
    if counts == count_off
      # set the dequeued element to be 0, i.e. it has been counted out
        player_array[arr_index] = 0 
        counts = 0

        puts "The #{count_out}th dequeueing: #{arr_index}"

      arr_index += 1

    #   This makes the array to be circle.
    arr_index = 0 if arr_index == players

前幾日那個『貓的體重』的題目，幸得Ellena的評論提醒發現了一處錯誤。

因此公比(k1+k2)<1應該為|k1+k2|<1。
加上貓的體重允許負數，因此判斷無窮生長的條件更加複雜一些。將判斷方法略做修改。
不得不說，體重允許負數這個條件真是比想象中噁心。

代碼新改動了判斷貓的體重增長的方法weightGrow()，根據以下公比的不同情況進行判斷。
然後為了代碼的美觀，將成長天數遞增的循環單獨寫了一個方法 int dayPlus(int dayin)。新的weightGrow()各種條件語句的嵌套和層層縮進已經快要觸發我的強迫症因此我決定稍微減少一下嵌套和縮進。

關於等比數列公比的內容看這裏：Geometirc Series#Common Ratio

要使貓的體重無窮增長，且因為有體重限制(upper bound)，則要使等比數列totalWeight(i + 1) = (k1+k2) * totalWeight(i)無窮遞縮。由於有體重限制和負體重的情況，這個等比數列和普通無窮等比數列的不同就是，不討論收斂(converge)和發散(diverge)以及極限(limit)的情況。
換言之，逐日的體重能無窮遞縮就行。
由於貓的體重可以為負數，在第一天的初始體重weight(1) ≤ k的大前提下（否則第一天就不會繼續生長，立刻停止）要分為三種情況來討論公比的各種情況。（等我再去扒拉一張草稿紙來……

第一種情況：k >0 這個屬於一般情況，無窮遞縮數列種所有項ai∈R且所有ai的和不大於體重限制k。（按照這隻貓的尿性體重為無理數應該也是妥妥的沒問題）
因此公比r = (k1 + k2)分以下情況：

  • |r| < 1 i.e. -1 < r < 1 數列各項無限向0收斂。inf。
  • r > 1 當weight(1) > 0時，體重最終超過限制k，停止生長。當weight(1) < 0時，數值向負無窮發散。inf。
  • r < -1 體重在正負間振盪並最終超過限制k。停止生長。
  • r = 1 奇葩的情況其一，體重會無限地保持初始值。由於有weight(1)≤k的大前提，這個該算是已經停止生長了呢，還是算是無限呢……這裏視作inf。

  • r = -1 奇葩的情況其二。體重會在±firstDayWeight之間振盪（oscillation，想了很久不知道這個詞在中文裏怎麼說，感謝123君指點，最終敲定了振盪這個詞），但是由於有大前提。inf。
    第二種情況：k ≤ 0 and weight(1) < 0 很容易引起問題的一個情況。特別特別討厭……需要一些仔細的考慮。
    因此公比r = (k1 + k2)分以下情況：

  • -1 < r < 0 雖然會向0收斂但是因為負負得正，所以會超過限制k。停止生長。

  • 0 < r < 1 陷阱。雖然數列無限向0收斂，但是要視k的情況而定。
    當k < 0時，停止生長。
    當k = 0時，inf。
  • r > 1向負無窮發散。inf。

Problem Description

威威猫不是一只普通的猫，普通的猫喜欢吃鱼，但威威猫最喜欢吃鸡腿。他每天都在不停的吃啊吃，吃了一只又一只鸡腿。现在他遇到了一个难题，如果他的体重太胖那么他的主人就不给他吃鸡腿了，所以他需要你的帮助。

威威猫的身体由n个器官构成，由于他的身体很特殊所以他的增长也很特殊（不要问为什么，喜欢吃鸡腿的猫已经够奇怪了）。他的增长有个k1和k2系数，而且每天的增长量和前一天有关，我们假设这n个器官在第i天的数值分别是a(i,1), a(i,2), a(i,3) …… a(i,n)，那么，第i+1天他每个器官的数值就会变成:

a(i+1,1) = k1 a(i,1) + k2 a(i,2)

a(i+1,2) = k1 a(i,2) + k2 a(i,3)

……

a(i+1,n) = k1 a(i,n) + k2 a(i,1)

威威猫的体重等于他的所有器官的数值之和，并且他还拥有一个特殊的机能，就是会自动检测自己的体重，如果他的体重比K大，那么就会自动停止生长（为了每天都能吃到鸡腿），由于威威猫的特殊身体构造他的体重是可能会变成负数的。（本來是想不應該吐槽題目內容的但是這個體重負數真的瞬間讓我感覺到了這隻貓在物理和物質上的逆天艾瑪忍不住了……

现在我给你n个器官的初始数值和他的增长系数k1,k2，请问他几天之后会停止生长，如果他永远无法停止生长那么就输出”inf”。(引号不用输出)
核心的計算就在於用遞推推導第i+1天的體重和第i天的關係，小難點是如何判斷永遠無法停止生長。
設總體重函數為K(i)，其中i表示天數且i≥1。則求和為

K(i+1)=∑(t=1,n)k1a(i,t)+k2a(i,t)=∑(t=1,n)(k1+k2)*a(i,t)

沒有數學公式插件略麻煩就這樣將就吧，推導並不難，仔細觀察一下就知道。總結一下這條式子其實就是totalWeight(i+1)=(k1+k2)*totalWeight(i)
由於體重能負增長，因此(k1+k2)可以小於1或者為負數。因此當(k1+k2)<1並且(k1+k2)<1的時候，它就永遠無法停止生長（永遠負增長）。

Input

输入数据第一行是一个正整数T，表示有T组测试数据；

每组数据的第一行包含4个数字n,k1,k2,k，代表威威猫有n个器官，他的生长系数是k1,k2，当体重超过k的时候他就停止生长。

接下来的一行是n个数ai，代表威威猫每个器官第一天的数值是多少。

[Technical Specification]

T ≤100

1 ≤ n ≤ 10000

-100 ≤ k1, k2 ≤ 100

1 ≤ k ≤ 10^18

1 ≤ ai ≤ 1000（1 ≤ i ≤ n）

import java.io.*;
import java.util.ArrayList;

public class Cat{
private int n; //Quantities of organs
private int t; //Number of groups of test data
private float k1,k2; //Coefficients
private float k; //Given upper bound weights
private float firstDayWeight; //Weight on the 1st day
private float totalWeight; //Total weight by the sum of each organ
ArrayList out = new ArrayList(); //output string list

//Cat's weight on the day d.
private float weight(int d){
    totalWeight = firstDayWeight;
    for(int i = 1; i &lt; d; i++)
        totalWeight *= (k1 + k2);

    return totalWeight;
}

public String weightGrow(){
    int day = 1;
    if (weight(1) &lt;= k) {
        if((k1 + k2) &lt;= 1)
            return &quot;inf&quot;;
        else{
            while (weight(day) &lt;= k)
                day++;
        }
    }
    if (day == 1)
        return Integer.valueOf(day).toString();
    else
        return Integer.valueOf(day - 1).toString();

}
